# Link-ual Log: An Agentic Workflow to Build Personal Dictionary for Language Aquisition

## Overall Preface

### Target User

This project is built with Ubuntu. The basic environment management includes `npm`,`uv`. You should know how they work in advance.

If you aren't equipped with basic knowleadge, the setup maybe a bit challenging to you. But I believe they are useful for the target users.

If you are:

- Almost an undergraduate students with computer-related background
- Have experience building a personal static website (especially material-for-mkdocs on gitpages)
- Have demand to learn English with technology in a more sustainable way 

This project provides you 

- a browser-plugin, enhancing your experience on video websites; 
- a master-server, collecting data from your phone (especially the picture you want to OCR and organize into structure), also from the browser-plugin; 
- a second-server, give a toy application for memorizing; 
- a website-template, give an implementation of static website

### Quick Start?

The project contains different modules which can work individually for different demands. Checking "For User" under each "MODULE-NAME Set Up" provides you a relatively quick start.

However, I've done a Makefile for myself, to make the workflow entrance simpler. Learn what Makefile is and take advantage of it.

## Motivation

I personally maintain a knowledge base of English learning, deployed on a static website. Even with the help of LLM, the repetitive workflow that:

1. Transform the data from non-text into text
2. Open LLM website, copy and paste the raw source, chat with it
3. Create or find the file
4. Combine the contents, give tags and cite

makes the aquisition not as productive as I wish. So I develop a system to make the experience more intuitive and sustainable:

1. Focus Splitting. I wish I can independently do a job of English without the system for a long time. After that, I will send the source to the system. The system will automatically do most of the dirty work.
2. Storage Friendly. We don't want too much non-text source (whose info is just about text) taking up large memory in our device. So they should be aborted after committing.
3. Review & Merge Friendly. The repetitive job is agentic, so I must check the quality by hand, which serve as an important step of language aquisition. And there is also a problem of merging to an existing file. This by-hand section is left deliberately. But I also want it to be as easy as possible.

---

P.S. 'Link-ual' sounds like 'lingual'. The output of the system is a 'lingual log' for what we just learned.

### Data Aquisition

This project requires a LAN(local area network) environment. This provides convenience for data transmission. Like you can use PC as master server to host the backend, collect photos by your phone and send it from the frontend.

In details, I design 2 main ways to collect data:

1. You can use the frontend hosted in LAN to send text or pictures to the server. This is mainly for image materials. (You can set the name of images if you want. This will be helpful for future citation. )
2. You can use a browser-plugin to send subtitle on video website and corresponding prompt to the server. This is mainly for video materials. (The citation info is automatical. )

### Data Processing

For image material, it will try to OCR the content, and defaultly try to give the key word and its context. This step is under human supervised for a better performance. 

- If the OCR content is bad, you can try to re-OCR
- If the OCR content is OK, you can delete the recommendation or select the sentence and words yourself.

The recommondation algorithm, on one hand, if you have marked underline in image material, is generated by OCR & LLM processing; on the other hand, we will provide a simple RAG method, retrivie the dictionary of your customized list (The bottom model is NOT vector database but simple string matching powered by AC-automaton). 

Then this step will transform the source materials into json. They are likely to be named as a single word, or a phrase connecting with `-`. e.g. `run.json`, `in-a-long-run.json`.

The json cites the origin for the video, or the name if you set, and contains the date.

### Merge and Review

You give and save the directory of database in master-server. Then this step will try to find the file in your database to decide merge or create. You will be provided a Git-style comparison view.

### Enjoy or Deploy

I design 2 main ways to take adavantage of the result:

1. Second-server gives an instance to use json to build a RECITE WORDS application by hosting it in LAN. 
2. Take my website as an instance. The result should be transformed into markdown. So we can hook a program automatically if you want.

## Browser-plugin Set Up

### Preface

This is a GMonkey Browser Plug-in for Subtitle Enhancement. 

Firstly you should learn what the GreaseMonkey(油猴) is, and its relationship with TamperMonkey(篡改猴). I'd like to call this type of browser plug-in to be GMonkey, just like the uniform API 'GM' they provide. 

### For User

If you just want to try this plug-in, you may ONLY need the file `browser-plugin/user/linkualog.user.js`.

Create a new script in your TamperMonkey, copy the whole content from the file into it, save it (`Ctrl + S`). Maybe there're some permissions required.

- Enjoy your experience in `youtube.com` with sidebar. When you watch a video with subtitles, turn the subtitles on until the sidebar recieves the data.
- Send the pieces of subtitle and corresponding prompt (or left blank) to the hoster by local area network, for further processing.

### For Devloper

If you want to get a devloper vision of this project, you can check `browser-plugin/dev/`. Switch to this path in your terminal.

This part is built upon `Vite`(`vite-plugin-monkey`) and `React`. So just like usual projects, use `npm install` to generate `node_modules` which is gitignored.

Then you can use `npm run dev` to have a HMR experience. Open the localhost link in the browser with TamperMonkey, click the given botton `install the plugin`. If nothing happened to your website, you can try a CSP-unblock plugin to unblock the restriction of CSP.

If you want to pack up the `user.js` yourself, use `npm run build`.
